LABORATORY  L6  : C-arrays + array algorithms (part II)

Introductory description:

Basic array processing algorithms (fill, compare items, search, move items, delete, add/insert, . . . ).

Materials for laboratory self preparation:
  • W3schools tutorials - C Array exercises with solution
  • W3schools - Array exercise (6) - Print all unique elements of an array

Note: As a possible improvement, the currently used number of elements of array (logical size), could be different/smaller than real size N. Usually it can be represented by additional variable n.
In this case, there are two sizes:
1) physical/maximal size N  (constant)
2) logical/used size n           (variable)

   const int N=100;    // maximal capacity
   double array[N];   // fixed-size array
   int n=0;                  // currently used "logical" size

In all tasks of Laboratory_6, please use such quasi-dynamic array with additional "logical" size.
(Wiki: Bounded-size dynamic arrays),

TASK L6.T1

Description of the task to be performed:

Write a program that fills n-elemental array with random integers from range [ min, max ],
(where:  n, min  and  max  values ​​are read from the computer keyboard)
and then checks:

a) if all numbers are arranged in ascending order  ( i.e.  arr[0] < arr[1] < arr[2] < . . .  )

b) whether there is at least one repeated value (any repetition:  2, 3, 5, 2, -7, . . . )

The rand() function, from library <stdlib.h>, can be used to generate random numbers.

Functions  srand() and time()  
(the second is from library <time.h>),
can be used to initialise random pseudo-generator.

Description of the task:

a) first will load the data (from keyboard) to n-element array of short integers,
